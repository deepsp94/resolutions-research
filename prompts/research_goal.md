# Codex Goal: Prediction Market Failures: Research on resolution, rules, "additional context" by Polymarket, and other such failures

## Goal Text

Research and document the prediction-market resolution/rules failures named in `initial_research.md`, producing a separate sourced Markdown case file for each incident under `markets/`.

For each case, use only the incidents already named in `initial_research.md` as the source list. Do not add new platform failures unless the user explicitly expands scope. Use internet research, primary sources, contemporaneous reporting, on-chain/API records, and Exa (API key in .env file) if ordinary browsing is blocked or insufficient.

Each case file will go into a public website in an effort to publicly explain the mistakes made by platforms, in each case that the platforms did make a mistake. It should identify the market, rules, final result, allegation, platform's explicit or strongest implicit defense, strongest arguments on both sides, and a provisional or conclusive assessment. Every material factual claim must be grounded in cited sources. If the evidence is insufficient, state the blocker precisely rather than overstating the verdict.

The goal is complete when every incident from `initial_research.md` has either a sourced case file explaining whether and how the platform made a mistake, or a sourced blocker note explaining why the incident cannot be evaluated conclusively. Do not assume every seed allegation is a real platform error.

## Success Criteria

- One Markdown file exists for each sufficiently identifiable market, named consistently under `markets/`.
- Each file contains sourced facts, both-side arguments, the platform defense, and a careful assessment of whether and how the venue erred.
- The work does not introduce incidents outside `initial_research.md`.
- Weak, Reddit-only, vague, or unidentified incidents are marked as blockers rather than treated as proven failures.
